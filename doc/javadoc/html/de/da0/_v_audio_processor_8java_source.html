20 /**
21  * Base class for all modules processing audio samples.
22  *
23  * An audio processor can have any number of {@link VAudioPort_Input inputs}
24  * and any number of {@link VAudioPort_Output outputs}.
25  *
26  * Mostly, an audio processor receives sample data from its input ports,
27  * does some work with the sample data and pushes the result to its
28  * output ports. Some audio processors only receive data
29  * (e.g. {@link VFrontend}), some others only send data
30  * (e.g. {@link VAudioDriver}).
31  *
32  * Processing is normally driven from the input side - whenever data is
33  * pushed to the inputs, they will push the data on to the processor
34  * which will then do the processing with the data and then push the
35  * result to its outputs.
36  * There are exceptions, for example an {@link VAudioDriver audio driver}
37  * receives sample data from an external source and pushes the data on
38  * to its outputs at the appropriate time.
39  *
40  * This class does all the admin related to input and output ports. Port
41  * objects can be accessed by index or by key. Index based access is useful
42  * a client wishes to enumerate all ports.
43  *
44  * @author Peter Hoppe
45  */
