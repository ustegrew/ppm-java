void _ProcessInputs ( JJackAudioEvent  e)
private

Processes data ready to be sent to the Jack driver i.e.

towards the JackD server). This sounds paradox, but we define ports as INPUTS which receive data from our program and send it to the JackD server.

We use a very crude overflow protection - this saves us lots of code which has to deal with exceptions, resp. loop over the buffered samples etc. This is time critical data handling, therefore we want to keep it as simple as possible.

The client program is responsible to ensure that data frames don't exceed the prescribed buffer length.

Parameters
eThe handle allowing access to the input buffers.
