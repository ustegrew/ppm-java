132  void Receive (float sample)
133  {

136  TNodeConverterDb_Endpoint_Out out;
137 
138  if (sample >= gkMinThreshold) /* Re calculations: [100] */
139  {
140  x = 20 * Math.log10 (sample);
141  }
142  else
143  {
144  x = gkMinDB;
145  }
146 
147  dB = (float) x; /* [110] */
148 
149  fStats.SetValue (sample);
150  fStats.SetDB (dB);
151 
152  out = (TNodeConverterDb_Endpoint_Out) GetPortOut (0);
153  out.PushSample (dB);

157 /*
158 [100] Peak values, Raw values, Calibration and all that...
159 
160  For each sample, JJack delivers a floating point value
161  in the range [-1.0, 1.0] floating point units.
162  For each frame, we map all negative values to the
163  positive range, effectively rectifying the signal.
164  The positive maximum value in each frame becomes the
165  peak value.
166 
167  Peak value vs. Input signal voltage
168  -----------------------------------
169  Our calculations make the simplifying assumption
170  that the raw values delivered by jackd map directly
171  to voltage, e.g. 0.5 means: 0.5V. It's an arbitrary
172  assumption; in reality the voltage will differ wildly
173  from sound card to sound card. That's why we can't
174  use a unified calibration procedure. Future developments
175  could include an input gain setting that calibrates the
176  PPM to map peak value 1.0 against PPM 7 + 4dB (i.e.
177  an imaginary PPM 8) to add some headroom.
178  This is a very crude measurement. For now, if the sound
179  card input is connected to a mixer output, then use the
180  mixer's VU meter and match +4dB to PPM 7.
181 
182  Calculations
183  ------------
184  Range (peak value, fpU): floating point units, from JJack provided values
185  [0.0, 1.0]
186 
187  Generic formula: ratio (vdB): Signal Voltage (v) against reference voltage (vr):
188 
189  vdB = 20 * log (v / vr)
190 
191  Let vr = 1.0, then
192  vdB = 20 * log (v)
193 
194  Our mapping fpU -> dB:
195  fpU = [0.0, 10^-7.5]: -130
196  ]10^-7.5, 1]: -130 .. 0
197 
198  Note that
199  -130dB: Hearing threshold
200 
201  which computes to a voltage of:
202  10 ^ (-130/20) = 10^-7.5 = 3.16 * 10^-8
203 
204  Calculating peak values in dB:
205  for p: [0, 3.16E-08]
206  pdB = -130
207  for p: ]3.16E-08, 1.0]
208  pdB = 20 * log (p)
209 
210 [110] There are situations where downcasting from double to float
211  is dangerous. Here it's fairly safe:
212  * We deal with dB values, i.e. the really interesting part
213  is to the left of the decimal point. To the human ear,
214  -60dB isn't noticably different from -60.2dB.
215  * Round down errors (double -> float) happen in a dimension
216  somewhere below 1E-30 which makes no difference when
217  displayed on a meter display.
218  * We produce a use-once value, i.e. we won't use this value
219  as a base for the next peak calculations. Therefore we won't
220  have accumulating errors.
221 
222 */
