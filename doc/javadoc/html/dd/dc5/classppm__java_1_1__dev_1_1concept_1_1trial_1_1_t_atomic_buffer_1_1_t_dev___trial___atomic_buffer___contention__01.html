Motivation: Find out whether the mutex free synchronization in TAtomicBuffer is working. If we get corrupted data we have a race condition problem, due to faulty synchronization technique.

Assumption: We won't test whether AtomicInteger is truly atomic - all documentation indicates an AtomicInteger is atomic across multiple threads using it, therefore we just follow the documentation and presume that it is so. Result: Trial showed that data was never corrupted, even in heavily contended situations. Therefore our synchronization with atomic integer does work.

This crude break trial wouldn't work with the present version of TAtomicBuffer. The present version won't block the high priority thread when the low priority thread gets blocked.
