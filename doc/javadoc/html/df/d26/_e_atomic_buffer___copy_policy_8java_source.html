20 /**
21  * The Copy policy. To be supplied as argument to the constructor.
22  * <dl>
23  * <dt>{@link #kCopyOnGet}</dt>
24  * <dd>
25  * Data will be deep copied when the consumer thread retrieves it.
26  * This means that the data won't be copied upon setting it
27  * ({@link TAtomicBuffer#Set(FloatBuffer)}) which
28  * means that setting data costs the least time. Use this policy
29  * when the producer thread has a higher priority than the consumer thread.
30  * </dd>
31  * <dt>{@link #kCopyOnSet}</dt>
32  * <dd>
33  * (<b>default</b>). Data will be deep copied when the producer thread sets it.
34  * This means that the data won't be copied upon getting it
35  * ({@link TAtomicBuffer#Get()) which means that getting the data
36  * costs the least time. Use this policy when the consumer thread has
37  * a higher priority than the producer thread.
38  * </dd>
39  * <dt>{@link #kNoCopy}</dt>
40  * <dd>
41  * Data won't be copied, but we pass on the original data frame objects.
42  * This policy is risky and should only ever be used if either the producer
43  * or the consumer guarantee their own data copy. This policy is necessary
44  * in some situations to prevent unnecessary multiple copying of data frames
45  * (e.g. when data is passed from one module to another).
46  * </dd>
47  * </dl>
48  */
49 public enum EAtomicBuffer_CopyPolicy
50 {
51  kCopyOnGet,
52  kCopyOnSet,
53  kNoCopy
54 }
